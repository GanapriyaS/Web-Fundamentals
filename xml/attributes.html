XML Attributes

  • XML elements can have attributes, just like HTML.
  • Attributes are designed to contain data related to a specific element.
  • Attribute values must always be quoted. Either single or double quotes can be used.
    e.g) For a person's gender, <person gender="female"> (or) <person gender='female'>
  • If the attribute value itself contains double quotes you can use single quotes
    e.g) <gangster name='George "Shotgun" Ziegler'></gangster> (or) <gangster name="George & quot;Shotgun& quot; Ziegler"><ds/gangster> (character entities)

Cons

  • attributes cannot contain multiple values (elements can)
  • attributes cannot contain tree structures (elements can)
  • attributes are not easily expandable (for future changes)

There are no rules about when to use attributes or when to use elements in XML.

3 Ways
<note date="2008-01-10">
    <to>Tove</to>
    <from>Jani</from>

<note >

metadata (data about data) should be stored as attributes, and the data itself should be stored as elements.
