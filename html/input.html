<input>

  1. The < input > tag specifies an input field where the user can enter data.
  2. The < input > element is the most important form element.
  3. The < input > element can be displayed in several ways, depending on the type attribute.
text

defines a single-line text field.

The default width of the text field is 20 characters.

password

Defines a password field (characters are masked).

button

The type attribute specifies the type of button

Submit or request

  • submit - The button is a submit button (submits form-data)
  • reset - The button is a reset button (resets the form-data to its initial values)

checkbox
  • Defines a checkbox.
  • The checkbox is shown as a square box that is ticked (checked) when activated.
  • Checkboxes are used to let a user select one or more options of a limited number of choices.
radio
  • Defines a radio button.
  • Radio buttons are normally presented in radio groups (a collection of radio buttons describing a set of related options). Only one radio button in a group can be selected at the same time.
  • Note: The radio group must have share the same name to be treated as a group. Once the radio group is created, selecting any radio button in that group automatically deselects any other selected radio button in the same group.
date
  • Defines a date picker.
  • The resulting value includes the year, month, and day in the format mm/dd/yyyy
hidden
  • Defines a hidden input field.
  • A hidden field let web developers include data that cannot be seen or modified by users when a form is submitted.
  • A hidden field often stores what database record that needs to be updated when the form is submitted.
